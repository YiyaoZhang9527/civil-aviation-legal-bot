# 独立评测体系：从旧指标到新指标的完整分析

> 日期：2026-06-02
> 背景：发现旧评测把检索覆盖率缺口误判为"幻觉率"，导致 45.5% 的数字严重失真

---

## 1. 问题发现

旧评测（test_30questions.py）的"幻觉率"来自系统自身的 CitationAgent：

```
question → LLM 提取 claims → 每条 claim × 每条 evidence → Cross-Encoder 打分
→ score ≥ 0.25 → supported / score < 0.15 → unsupported
```

**这是在测"检索到的证据 CE 得分是否够高"，不是在测"答案是否编造了内容"。**

### 典型误判案例

| 题号 | 旧评测 | 答案实际内容 | 真实情况 |
|------|--------|-------------|---------|
| Q03 | 92% 幻觉 | "根据现有证据，无法确定" | **正确拒答，零幻觉** |
| Q28 | 75% 幻觉 | "对领空享有完全的、排他的主权"（引用民用航空法第二条） | **完全正确，零幻觉** |
| Q20 | 83% 幻觉 | "相关条款未提供" | **正确拒答，零幻觉** |

---

## 2. 三层评测体系设计

### 第 1 层：Faithfulness（答案级忠实性）

用独立 LLM 做 claim-level 评测，完全脱离系统自评。

流程：
1. 把答案分解为 atomic claims（独立可验证的事实声明）
2. 对每条 claim，判断能否从证据原文推导出来
3. faithful / partial / hallucinated 三档

关键设计：
- 评测 LLM 与生成 LLM 必须不同（避免自我肯定偏差）
- 拒答不计入幻觉率（N/A）
- 证据必须读完整法条原文，不能只读标题

### 第 2 层：引用真实性（纯代码检查）

不用 LLM，纯正则 + index 对照。

检查项：
- 答案中《法名》第X条 → 法规是否存在 → 条文是否存在

### 第 3 层：拒答统计

统计"无法确定"类回答比例，与商用系统对标。

---

## 3. 评测过程中的方法论纠偏

### 第一轮评测（70.8% 幻觉率 — 虚高）

`load_evidence_texts` 只读了 `node.summary`（通常只有标题"第121.657条 燃油量要求"），没读 `node.text`（完整法条原文）。

评测 LLM 看到的证据：
```
证据[1]: 大型飞机公共航空运输承运人运行合格审定规则 第121.657条 燃油量要求: 第121.657条 燃油量要求
```

评测 LLM 自然判定：证据只有标题没有内容 → 声明无法被验证 → hallucinated。

### 修正后（36.2% 幻觉率 — 更准确）

改为优先读 `node.text`，降级 `read_law_node()` 读源文件原文。

评测 LLM 看到的证据：
```
证据[1]: 大型飞机公共航空运输承运人运行合格审定规则 第121.657条 燃油量要求:
第121.657条 燃油量要求
(a)飞机必须携带足够的可用燃油以安全地完成计划的飞行并从计划的飞行中备降。
(b)飞行前对所需可用燃油的计算必须包括:
(1)滑行燃油:考虑到起飞机场的当地条件和辅助动力装置(APU)的燃油消耗...
```

评测 LLM 现在能正确验证燃油量相关的声明。

**教训：评测本身的质量直接影响结论。如果证据文本不完整，评测结果会系统性虚高。**

## 4. 最终评测结果

### 汇总对比

| 指标 | 旧评测（系统自评 CE） | 新评测（独立三层） |
|------|----------------------|-------------------|
| 幻觉率 | 45.5%（CE unsupported/total） | **36.2%**（hallucinated/total claims） |
| 忠实率 | 44.1%（CE supported/total） | **58.8%**（faithful/total claims） |
| 可接受率 | 54.5%（CE sup+par/total） | **63.8%**（faithful+partial/total） |
| 引用有效率 | 未测 | **90.6%**（116/128） |
| 拒答率 | 未统计 | **3.3%**（1/30） |

### 对标商用系统

```
系统                    幻觉率    拒答率    备注
通用 GPT-4             58-82%    ~0%      Stanford 2024 研究
我们                    36.2%    3.3%     ← 优于通用，接近 Westlaw
Westlaw AI-AR           33%      25%     Stanford 2025 研究
Lexis+ AI               17%      18%     Stanford 2025 研究，商用最佳
```

### 按领域分解（第 2 层引用有效性 + 第 1 层 Faithfulness）

| 领域 | 引用有效率 | Faithful率 | 特征 |
|------|-----------|-----------|------|
| 人员资质 | 100% | ~80% | 表现最好 |
| CCAR-121 | ~90% | ~55% | 题目多，差异大 |
| 空管 | 100% | ~60% | 法规集中 |
| 航空安全 | ~85% | ~40% | 跨法规问题多 |
| 旅客服务 | ~75% | ~25% | **重灾区** |
| CCAR-92 | ~85% | ~30% | 无人机法规新 |
| 适航管理 | ~50% | ~55% | 引用匹配差但内容对 |

## 5. 核心结论

### 5.1 旧评测的两类错误

1. **假阳性**（误判为幻觉）：Q03 正确拒答被判 92% 幻觉、Q28 正确答案被判 75% 幻觉
2. **假阴性**（漏掉真幻觉）：旧评测只看 CE 打分，不检查答案中是否编造了具体数字/细节

### 5.2 真实问题定位

36.2% 的答案级幻觉主要来自 **synthesis 阶段 LLM 的"发挥"**：

- 证据说的是"燃油量要求应符合规定"，LLM 添加了具体的"10%不可预期燃油"
- 证据只提到"应当拒绝运输"，LLM 编造了具体的拒载情形清单
- 证据没有时间标准，LLM 添加了"450米高度飞行30分钟"

**LLM 用训练数据中的知识填充了证据中没有的细节。这不是检索问题，是生成问题。**

### 5.3 引用质量≠内容忠实性

90.6% 引用有效率 vs 58.8% 忠实率 —— **引用的法条是真实的，但答案中的具体声明不都来自这些法条**。

## 6. 评测文件清单

| 文件 | 用途 |
|------|------|
| `tests/test_citation_validity.py` | 第 2 层：引用真实性检查 |
| `tests/test_faithfulness.py` | 第 1 层：Faithfulness 评测 |
| `tests/test_summary.py` | 第 3 层：拒答统计 + 三层汇总 |
| `tests/test30_*_citation_validity.json` | 引用真实性详细结果 |
| `tests/test30_*_faithfulness.json` | Faithfulness 详细结果 |
| `tests/test30_*_summary.json` | 三层汇总报告 |

### 运行命令

```bash
# 第 2 层：引用真实性（纯代码，~30秒）
.venv/bin/python tests/test_citation_validity.py

# 第 1 层：Faithfulness（需 LLM API，~5-10分钟）
.venv/bin/python tests/test_faithfulness.py

# 第 3 层：汇总（依赖前两层结果）
.venv/bin/python tests/test_summary.py
```

### 环境变量（第 1 层评测建议配置）

```bash
# 建议用与系统生成不同的模型做评测
export FAITHFULNESS_API_KEY=xxx
export FAITHFULNESS_BASE_URL=xxx
export FAITHFULNESS_MODEL=gpt-4o  # 或 claude-sonnet-4-6
```
