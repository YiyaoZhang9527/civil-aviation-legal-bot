# 30题测试全失败 — 根因分析

**时间:** 2026-06-01
**现象:** 30题全部返回 "抱歉，处理您的问题时遇到了异常"，evidence_count=0

## 根因：双重过滤致死

### 故障链（4步）

1. **向量召回偏差** — `gte-large-zh` 把 "燃油排泄和排气排出物规定"（环保法规）排到前列，而不是真正相关的 CCAR-121。query "飞机起飞前需要满足哪些燃油量要求" 被错误路由到环保法规。

2. **条级文本空洞** — 检索到的候选条文 summary 只有标题（如 `"第34.31条 排气排出物标准"`），`parser.py:63` 的 `_summary_from_text` 只取前120字，很多条文原始文本很短。

3. **Cross-Encoder 精排全部过滤** — `Cross-Engine精排: 10条 → 0条 (阈值>=0.1)`。候选证据的 rerank 分数全部低于 `RERANKER_MIN_SCORE=0.1`，被全部干掉。

4. **空证据 → 异常** — `retrieval_agent.retrieve()` 返回空列表，后续 `citation_agent` 和 `synthesis_agent` 处理空输入导致异常，被 `LegalOrchestrator.answer()` 的 try/except 捕获，返回 "抱歉，处理您的问题时遇到了异常"。

### 附加问题：进程卡死/极慢

- 最新运行 Q01 耗时 969 秒（16分钟），之前成功运行 Q01 只需 287 秒
- 原因：多进程竞争 CPU，且 sentence-transformers 未使用 GPU（GPU利用率仅 2%）
- 按 16 分钟/题计算，30 题需约 8 小时

## 关键文件

| 文件 | 问题点 |
|------|--------|
| `legalbot/config.py` | `RERANKER_MIN_SCORE = 0.1` 阈值过高 |
| `legalbot/retrieval.py:584-597, 707-720` | 精排过滤无 fallback，空结果直接透传 |
| `legalbot/tree_retrieval.py` | Level 0 向量召回精度差 |
| `legalbot/agents.py:775-783` | try/except 吞掉真正异常 |

## 修复建议

### 短期（立即止血）

1. **RERANKER_MIN_SCORE 改为 0.0** — 让 LLM 自行判断证据相关性，不要在精排阶段一刀切
2. **精排空结果 fallback** — 精排后为空时返回精排前候选，而非空列表

### 中期（根本改善）

3. **条级 summary 质量** — parser 中对条文提取更多文本（200-300字而非120字）
4. **向量模型升级** — `gte-large-zh` 对法律短文本区分度不够，考虑 `BAAI/bge-m3`
5. **Level 0 增加关键词路由权重** — 对 "CCAR-121" 等强信号 query 前置过滤无关法规
6. **GPU 加速** — 确保 sentence-transformers 使用 GPU 推理
