## 二、为什么改进了检索但指标下降了？

### 核心矛盾：RERANKER_MIN_SCORE 从 0.1 降到 0.05

降低 reranker 阈值的本意是让更多结果通过，避免适航指令那种"全被过滤"的情况。但副作用是：**大量低质量的证据也通过了**，这些证据在评测中被标为 unverifiable。

数据印证：
- 改进前：12 题有 12 条证据（EVIDENCE_LOAD_LIMIT=12）
- 改进后：部分题有更多低质量证据通过 reranker，导致 unverifiable 数量增加

### 拒答率上升：从 1 题到 3 题

| 新拒答 | 原因 |
|--------|------|
| Q02 燃油量 | 改进前给了部分答案（0/12 supported），改进后直接拒答 |
| Q19 拒载理由 | 改进前给了部分答案（0/12 supported），改进后直接拒答 |
| Q21 禁带物品 | 改进前给了部分答案（2/12 supported），改进后直接拒答 |

这是**正面变化**——Q02/Q19/Q21 在改进前给的答案本就几乎全是 unverifiable 的，改进后系统选择了拒答而非编造。

## 四、根因分析

### 问题 1：reranker 阈值降低引入噪声

`RERANKER_MIN_SCORE` 从 0.1 降到 0.05 后，更多低质量证据通过了过滤。这些证据在评测中标为 unverifiable，导致 unverifiable 率上升。

**解法**：不是简单回调阈值，而是需要更智能的过滤策略（如基于 query 类型动态调整）。

### 问题 2：完整管线的多 query 合并抵消了单 query 检索改进

单 query 测试时 Q13 能命中 CCAR-92 的 12 条结果，但完整管线中：
1. RewriteAgent 生成了多个 query
2. 不同 query 检索到不同法规的结果
3. 合并时非相关结果挤掉了相关结果

**解法**：改进多 query 合并策略（WRRF 或按法规分组去重）。

### 问题 3：拒答增加是正面信号但需要区分

Q02/Q19/Q21 的拒答说明系统在证据不足时更谨慎了——这比给 unverifiable 答案好。但 Q21 禁带物品的知识库已经有了，不该拒答。

**解法**：检查 Q21 为什么没检索到新增的禁限带目录。

| 下一步 | 修复多 query 合并策略，让单 query 的改善传导到完整管线 |

### 下一步优先级

1. **P0：多 query 合并策略** — 当前简单去重合并，需要法规分组或 WRRF 加权
2. **P0：reranker 动态阈值** — 不是固定 0.05，而是基于结果分布动态调整
3. **P1：Q21 禁带物品拒答排查** — 知识库已有但仍拒答
