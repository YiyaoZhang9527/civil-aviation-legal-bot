## 失败模式四分类

### 类目 1: Faithfulness 评测过严 (实际答案正确, 7道)
- 证据命中正确, LLM答案内容也正确, 但评测按"证据细节未明说"判unverifiable
- 主要是法规条款内容正确, 但评测要求"具体细节必须出现在evidence文本中"

| 题号 | 问题 | 命中证据 | 实际答案 |
|------|------|----------|----------|
| Q01 | 不在《运行规范》备降 | 121.25 + 91.103 (对) | 正确 |
| Q46 | 无人机事故后执照处理 | 92.1005/1003/1009 (对) | 正确 |
| Q69 | 零部件证件 | 21.303/21.311 PMA (对) | 正确 |
| Q74 | 飞机尾气排放 | 34.31/34.43/34.89 (对) | 正确 |
| Q83 | 导航设备停机 | 第60章 + 校验规定 (对) | 正确 |
| Q89 | 飞行签派员 | 65.7/65.13 (对) | 正确 |
| Q93 | 模拟机合格证 | 60.19/60.21 (对) | 正确 |

**根因**: deepseek-v4-flash 评测 LLM 严格按 evidence 原文逐字对应, 答案"内容正确但表述改写/概括"被大量判为 unverifiable

### 类目 2: 检索错配 (命中错误法规, 5道)
- 检索返回了相邻/同义但实际不匹配的法规, LLM 直接按证据作答, 答案跑题

| 题号 | 问题 | 错误证据 | 应有证据 |
|------|------|----------|----------|
| Q03 | 低于最低天气标准 | 91.103 (紧急偏离) | CCAR-121 运行最低标准专章 |
| Q10 | 特技飞行 | 21.214 (特许飞行) | CCAR-91 特技飞行专章 |
| Q13 | 无人机空域许可 | 民航法第十九条 (不相关) | CCAR-92 无人驾驶航空器空域 |
| Q27 | 飞行程序设计审批主体 | 第36/37/38条 (设计单位资质) | 应有"审批机关"专门条款 |
| **Q73** | **航空燃油适航审批** | **第八条/第五条 (无关)** | **应拒答: 燃油不在适航规章范围** |

**最严重案例 Q73**: 用户问"航空燃油是否需要适航审批", 证据给的是其他条款, LLM 强行套用 21.212"特许飞行证"作答, **答非所问**
**根因**: 关键词路由触发了完全不相关的子树 (Q10的"特许"被误读为"特技", Q73的"燃油"被混入"调机飞行")

### 类目 3: 检索缺口 (证据只给总则/前言, 5道)
- 检索命中了"同法规"但只返回了"适用范围/目的依据"等总则条款, 缺失具体要求条款
- LLM 知道证据不全, 但用通用知识/相邻条款补全答案

| 题号 | 问题 | 命中证据 (全总则) | 缺失 |
|------|------|-------------------|------|
| Q02 | 起飞前燃油量 | 121.1/121.3/121.5 总则 | CCAR-121.657 签派放行 |
| Q12 | CCAR-135飞行员资质 | 135.1-135.19 全是总则 | 135.243/135.244 资质要求 |
| Q19 | 拒载合法理由 | 各种不相关条款 | CCAR-121.535 拒绝运输 |
| Q49 | 通航乘客安保检查 | 第十条 机长培训 | 通用航空安保专章 |
| Q91 | 飞行员定期检查 | 67.21/67.23/67.65 体检 | 67.13 体检鉴定 + 附件A医学标准 |

**根因**: 三级树检索 Level 0→1 先验只传播到"部号", 没有精确路由到具体条款. CCAR-121 总则 (121.1, 121.3) 的文本总是最先被命中, 遮蔽了具体条款

### 类目 4: LLM 过度自信 (8道含hallucination)
- 证据/答案对不上, LLM 强行编造/过度泛化

| 题号 | 问题 | 编造内容 | 实际证据 |
|------|------|----------|----------|
| Q09 | 机长权力 | 第五十条"代理机长指派" | 实际是"所有人/承租人指派" |
| Q11 | 小型备降要求 | "IFR必须指定目的地备降" | 135.189(b) 允许特定条件下不选 |
| Q41 | SMS体系功能 | 135.5条有(d)款 | 实际只有(a)(b)(c)三款 |
| Q47 | 通航经营许可 | 民用航空法第三十四条 (化学产品) | 实际该条是无人机适航 |
| Q70 | 适航委任代表 | 第二条规定委任程序 | 实际第二条只规定适用范围 |
| Q80 | 空管事件报告 | "12小时内电话报告" | 证据是"立即电话报告" |
| Q86 | 雪情通告有效期 | "24小时最长" | 证据未明说具体时长 |
| **Q87** | **情报员执照** | **罚款500-1000元 + 2年禁令** | **实际200-1000元 + 无禁令** |

**最严重案例 Q87**: LLM 自信地给出"500元以上1000元以下罚款, 2年内不得申请", 实际证据是"200元以上1000元以下"且无"2年内禁令". 此类"编造具体数字+具体限制"是最危险的hallucination类型
**根因**: SynthesisAgent 的 prompt 鼓励"完整答案", 缺少"无证据不答"边界, 当细节缺失时倾向"用相邻数字填"

## 跨类目共性根因

1. **法规标题与内容错配**: 证据条款的"标题"和"实际内容"经常不匹配用户问题. 例如Q73的"航空燃油"被路由到"特许飞行"标题
2. **三级树Level 0→1 先验污染**: 总则条款 (121.1, 121.3, 91.103) 永远先命中, 屏蔽具体条款. Q02/Q12/Q19都是这个模式
3. **"完整答案"诱惑**: SynthesisAgent 倾向"补充完整", 缺乏"拒绝拼凑"机制
4. **数字/限制条件幻觉**: 几乎所有hallucinated声明都是"具体数字"或"具体时限". LLM 不会编造法理, 但会编造金额/期限/百分比

## 改进优先级建议

| 优先级 | 类目 | 改造点 | 预期影响 |
|--------|------|--------|----------|
| P0 | CAT4_细节编造 | SynthesisAgent 加"无证据拒绝编数字"硬约束 | 7条hallucinated中5条可消除 |
| P0 | CAT3_检索缺口 | 关键词路由细化 (从"部号"到"具体条款关键词") | 5道转为真正检索命中 |
| P1 | CAT2_检索错配 | 增加"问题意图-法规子章节"映射表 | 5道中至少3道可恢复 |
| P2 | CAT1_评测过严 | 调整 faithfulness LLM 的 prompt (允许"等价改写") | 7道正确题恢复 faithful 标记 |
