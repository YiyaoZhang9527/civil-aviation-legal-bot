## 四、F1 下降原因分析

| 变化 | 原因 |
|------|------|
| 检索缺口率 +12.7pp | Prompt 鼓励"覆盖"和"列举"，LLM 写出更多细节 claim，但部分细节在 evidence 中无直接原文支持 |
| 忠实率 -15.4pp | 同上 — claim 数从 ~250 增到 264 |
| 拒答率 -6.7pp | Prompt 鼓励"通读全文"+"列举"，LLM 极少判"证据不足" |
| 完整率 ≈ 不变 | 答案仍然覆盖用户问题 |

**本质权衡**：之前 LLM 保守 → 答案短但支持度高；现在 LLM 详细 → 答案长但 unverifiable 多。

## 五、可疑引用详情

| 题 | 类型 | 详情 |
|----|------|------|
| Q07 | 法规不存在 | `《第121.633条...》` — 这是条款号不是法规名 |
| Q08 | 法规不存在 | `《运输类飞机适航标准》(CCAR-25)` — LLM 推断 |
| Q12 | 条文不存在 | `《合格审定规则》（CCAR-61）第61.115条` — CCAR-61 不在索引 |
| Q18 | 法规不存在 | `《航班正常管理规定》` — LLM 推断 |
| Q21 | 法规不存在 | `《民航安全检查规则》` — LLM 推断 |
| Q24 | 法规不存在 | `《民用航空器适航指令规定》` — 实际叫"第8号" |
| Q26 | 法规不存在 | `《民航空管运行单位安全管理规定》` — LLM 推断 |

**根因**：LLM 根据条款内容"猜"了标准化的 CCAR 法规名（CCAR-25/61/95 等），但数据库用"原始文件名+第X号"格式。

**修复方向**：在 evidence 中给 LLM 提供"法律名 ↔ 索引文件名"对照表，让 LLM 复用索引中的精确名称。

## 九、下一步建议

1. **F1 回升策略**：在 prompt 中加"如果 claim 在 evidence 中找不到具体支撑，请合并或删除"，让 LLM 自动去重 unverifiable claims
2. **法规名规范**：在 evidence_index 中提供 CCAR 编号到索引文件名的映射
3. **提升 unsupported 区分度**：将 unverifiable 进一步拆为"合理推断"和"无依据猜测"，让用户看到更细的信号
4. **Re-rank 优化**：当 supported evidence 不足 3 条时，主动扩大检索范围
