## 关键洞察

### 1. 我们 2.8% 幻觉率 vs Lexis+ 17-33% → 表面领先 6-12 倍

**但这是误导。** Stanford 是 query-level（一次查询出一个错就算），我们是 claim-level（声明级）。
- Stanford 测的 17-33% 包含：法条不存在 + 引用错位 + 事实编造 + 推理错误
- 我们 2.8% 只计"证据明确矛盾"，41.4% unverifiable 算另一类

**若按 Stanford 口径换算**（把 unverifiable + hallucinated 合并为"用户可能被骗"）：
- 我们的"用户读到无证据支持内容"概率 = **44.2%**
- 比 Lexis+ AI 的 17-33% 反而**更高**

### 2. 真实差距在检索覆盖率（41.4% unverifiable）

这不是 LLM 问题，是**树检索召回不够**。Vectara 数据显示，专用 RAG pipeline + 强 reranker 可把 hallucination 压到 3% 以下，但前提是**检索覆盖率高**。

### 3. 任务难度完全不同

- Harvey 94.8% 在 Document Q&A：**给定文档，问其中事实** — 简单检索
- 我们的 52.8% Faithful：用户口语问题 → 多法规跨章节推理 → 合成答案
- **任务难度差一个数量级。** Harvey 没有"跨 129 部法规"这种问题。

### 4. 中文法律 RAG 整体落后英文 1-2 年

- 英文商用法律 AI（Harvey、Lexis+）有 5+ 年积累、海量标注、专有微调
- 中文法律 AI（ChatLaw、智谱）刚起步，公开数据少
- 我们面对的"129 部民航法规+口语问题"是**更难的子任务**

### 5. 同类项目里我们不算差

- 不考虑任务难度，我们的 2.8% 幻觉 + 94.1% 引用有效率在中文法律 RAG 圈是**中上水平**
- 主要问题：41.4% 检索缺口（用户拿到答案但证据不支撑）

## 优化方向（按 ROI 排序）

1. **提升检索召回（最高 ROI）** — 把 unverifiable 从 41.4% 压到 25% 以下
   - 路径：query expansion、混合检索权重调优、子问题拆分覆盖
2. **优化 LLM judge 校准** — 当前 Faithfulness 是相对值，需要更多样本验证稳定性
3. **增量式索引** — 新法规能及时入库（当前 `convert_to_txt.py` + `--build-index` 流程已就绪）
4. **针对 unverifiable 题做 case study** — 看是 query 改写问题、tree 剪枝过激、还是法律本身缺失
