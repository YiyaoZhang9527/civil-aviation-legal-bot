## 三、关键发现

### 1. 关键词路由是最大贡献者（+18pp）

激活 `_resolve_law_by_keywords()` 单独贡献 B (46%) vs A (28%) = +18pp。核心原因：

- **Q05 值勤期**: 0→11 (+11) — CCAR-121 被正确路由
- **Q22 安全信息报告**: 3→12 (+9) — 安全信息管理规定被路由
- **Q25 维修资质**: 4→12 (+8) — CCAR-145 被路由
- **Q01/Q08/Q11**: 各 +7 — CCAR-121 被路由

### 2. 通用条文降权与关键词路由有协同效应

单独降权（C: +6pp）效果有限，但与 KW 组合（D: +23pp）超过各自之和（B+C=24pp 但独立跑），说明两者互相增强。降权让路由注入的正确法规的**具体条款**更容易浮出。

### 3. D 是唯一零退步的组合

17 题进步、0 题退步。其他所有组合都有退步题：
- B: Q26 退步 -8（空管被其他法规挤掉）
- F: Q26 退步 -6
- G: Q21 退步 -4

### 4. law_routes.json 宽泛匹配引入噪声

E (KW+Rules, 36%) < B (KW_Routing, 46%)——补充路由规则反而降低了效果。原因是 `law_routes.json` 的 jieba token 匹配太宽泛，给很多不相关法规加了 hint boost。

### 5. 扩大 Level 0 是负收益

G (L0=50, 44%) < F (L0=30, 47%)——更多候选意味着更多噪声。

### 6. 幻觉率从 0.4% 上升到 2.8%

这是一个需要关注的副作用。改进前系统拒答了 3 题（Q02/Q19/Q21），现在只拒答 1 题（Q22）。被检索到的更多证据让 LLM 敢于回答，但部分答案超出了证据支撑范围。

具体看：8 条 hallucinated 中 Q28（领空权利）有 2 条，Q04（运行规范）有 2 条，Q25（维修资质）有 1 条——这些都是答案较长的题目。

## 五、未解决的问题

1. **Q13/Q14 无人机题仍然偏弱**：Q13 从 0→4 有改善但仍不够，Q14 从 0→2 改善有限。CCAR-92 虽然被路由了，但 hint boost (0.3) 可能不够推动其章节到 Level 1 前列。

2. **Q26 空管不稳定**：A=10, B=2, D=8, G=10——波动极大，说明此题对 LLM 随机性敏感。

3. **幻觉率上升**：从 0.4%→2.8%，需在生成层加强管控。

4. **LLM 随机性导致组间波动大**：同配置 E (36%) vs B (46%) 差距 10pp，说明单次测试的统计噪音很大，理想情况应跑 3 次取平均。

## 六、推荐配置

采用 D (KW+Generic) 配置：

```python
KEYWORD_ROUTING_ENABLED = True
TREE_GENERIC_ARTICLE_PENALTY = 0.5
```

并在 fallback 硬编码规则中保持精确匹配规则（22 条），但**不依赖 law_routes.json 的宽泛匹配**——后续考虑调整 law_routes.json 的匹配阈值或完全只用 fallback 规则。
