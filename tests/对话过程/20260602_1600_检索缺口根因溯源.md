# 检索缺口根因溯源报告

> 日期：2026-06-02
> 对象：test30_20260602_111235 全量评测，95 条 unverifiable claims
> 方法：逐题对比"知识库实际内容"vs"检索到的证据"vs"LLM 答案"，定位管线断点

---

## 根因一：Level 0 法规过滤丢了关键法规（影响 ~35 条 unverifiable）

**机制**：树检索 Level 0 用向量+BM25 对 129 部法规排序，只保留 top 30。小众/长尾法规在向量空间中与 query 语义距离远，排不进去。

### Q24 适航指令（7/7 全 unverifiable）

- 知识库中有 `民用航空器适航指令规定`（law_id=第8号），包含：
  - 第二章 适航指令
  - 第六条 概述："当中国民用航空局发现民用航空产品上存在...不安全情况，将以适航指令的形式加以指明"
  - 第七条 适航指令的颁发
- **检索结果：零命中**。12 条证据来自民航法总则、行政处罚规定、无线电管理规定等完全无关的法规
- **断点**：Level 0 向量搜索 "民用航空器适航指令是做什么的" 没有把"第8号"这个只有几条的小法规排进 top 30

### Q13 无人机空域许可（10/10 unverifiable）

- 知识库中有 `民用无人驾驶航空器运行安全管理规则`（216 nodes），包含：
  - 第二节 空域管理
  - 第92.511条 空域划设（管制空域 vs 适飞空域）
  - 第92.535条 飞行活动申请服务
- **检索结果**：零命中 CCAR-92 条文。12 条证据全来自民用航空法总则（第11/12/16/17/18/21/31/32条）
- **断点**：Level 0 可能排进了 CCAR-92，但 Level 1 章节过滤丢了"空域管理"章节（该章节在一个大文档中，可能被其他章节挤掉）

### Q23 事故调查（3/3 unverifiable）

- 知识库中有 `民用航空器事件技术调查规定`，包含完整的调查组织条款（第二章 调查的组织）
- 民用航空法有"第九章 搜寻援救和事故调查"
- **检索结果**：只命中了安全管理规定的一般性条款（信息报告、监督管理）
- **断点**：Level 0 没有把"事件调查规定"排进 top 30

## 根因二：证据文本截断/加载不完整（影响 ~25 条 unverifiable）

**机制**：`load_evidence_texts` 中 `text[:800]` 截断 + `read_law_node` 对长条文只读部分行。表格数据（markdown 表格）在截断中丢失。

### Q04 运行规范内容（10/10 unverifiable）

- 检索到了 `第121.25条 运行合格证及其运行规范的内容`（标题命中）
- 知识库中这条有完整内容：(a) 运行合格证 7 项 + (b) 运行规范至少 15 项
- **但 evidence 只加载了标题级内容**，条文具体列举项没有完整传入评测
- 证据只有 10 条（非 12 条），说明部分加载失败
- **断点**：`load_evidence_texts` 中 `text[:800]` 截断——121.25 条原文约 1500 字，截断后丢失了 (b) 款的具体列举

### Q05 值勤期限制（4/4 unverifiable，仅 3 条证据）

- 检索到了 `第121.485条 飞行机组的飞行值勤期限制`
- node.text 包含表格（表B 非扩编机组、表C 扩编机组），表格在 markdown 中是竖线分隔格式
- 只有 3 条证据加载成功（应该是 12 条）
- **断点**：表格数据在 `text[:800]` 截断中丢失，或者 `read_law_node` 行范围不够覆盖表格

### Q01 备降机场（5/5 unverifiable）

- 检索到了 `第121.641条 国际定期载客运行的目的地备降机场`
- 也检索到了 `第121.25条 运行合格证及其运行规范的内容`
- 但两者都是长条文，截断后丢失了关键细节
- **断点**：同 Q04，长条文截断

## 根因三：知识库本身缺内容（影响 ~13 条 unverifiable）

**机制**：法规正文只引用了外部文件（如"具体内容由民航局发布"），但外部文件不在知识库中。

### Q21 禁带物品清单（13/13 unverifiable）

- 知识库中有 `民用航空安全检查规则`（原文PDF版，105 nodes）
- 其中有 `第三十条`："旅客不得携带或者在行李中夹带民航禁止运输物品...**民航禁止运输物品、限制运输物品的具体内容由民航局制定并发布**"
- 但检索命中了第三条和第四条（概述性），没命中第三十条
- 即使命中第三十条，也只是说"由民航局发布"——**具体的禁带物品清单在民航局公告中，不在 129 部法规里**
- **双重断点**：
  1. 检索排序问题：第三十条排名低于第三条
  2. 知识库缺陷：缺少民航局禁限带物品公告

## 根因四：Level 2 条文排序/候选不够（影响 ~10 条 unverifiable）

**机制**：Level 2 候选上限 `TREE_ARTICLE_CANDIDATES=30`，相关条文在 30 名开外。

### Q07 签派员检查事项（3/3 unverifiable）

- 检索到了 `第121.531条 国内、国际定期载客运行的运行控制责任` 和 `第121.625条 签派员的职责`
- 但没检索到具体的"签派员放行前检查清单"条款
- 检索到了大量无关条文（121.663 备降机场、121.581 行李固定等）
- **断点**：query 改写偏了，把"签派员放行检查"改成了宽泛的"签派员"搜索，导致噪声条文排挤了核心条文

### Q09 机长权力（3/3 unverifiable）

- 检索到了民航法第 47 条相关的条文，但没检索到具体的"管束权"条款（第 46 条）
- **断点**：向量语义搜索把"机长在紧急情况下的权力"匹配到了"应急救援"而不是"管束权"

### Q29 所有权（6/6 unverifiable）

- 检索到了 `中华人民共和国民用航空器权利登记条例实施办法` 的部分条文（第4条、第19条）
- 但遗漏了所有权转让的核心条款
- **断点**：跨法规拼图不完整——需要同时检索民航法和权利登记条例的具体条文

## 汇总

| 根因 | 影响题数 | 影响条数 | 修复方向 |
|------|---------|---------|---------|
| **Level 0 丢了小众法规** | 3 题 | ~30 条 | 扩大 top_k 或加法规名 hint |
| **证据文本截断** | 3 题 | ~25 条 | 增大截断上限或按条文加载完整文本 |
| **知识库缺外部文件** | 1 题 | ~13 条 | 补充民航局公告等外部文件 |
| **Level 2 排序/候选** | 3 题 | ~10 条 | query 改写优化 + 精排增强 |
| **跨法规不完整** | 多题 | ~15 条 | query decomposition 跨法规 |

---

## 修复优先级

1. **P0：增大证据截断上限** — `text[:800]` → `text[:2000]` 或不截断。改动 1 行，直接修复 ~25 条 unverifiable
2. **P0：Level 0 保留更多法规** — `TREE_TOP_LAWS=30` → `40`，或对匹配 hint 的法规强制纳入
3. **P1：补充民航局公告到知识库** — 禁带物品清单等，一次补全 ~13 条
4. **P1：query 改写优化** — 减少"签派员检查"→"签派员"类的过度泛化
5. **P2：表格数据处理** — 对 markdown 表格做特殊处理，避免截断丢失
