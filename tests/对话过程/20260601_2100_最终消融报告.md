# 最终消融实验报告

> 日期: 2026-06-01
> 实验1: 8题x6组合 (基础消融)
> 实验2: 8题x9组合 (G组合 + 阈值扫描 + E_Full)

## 最终决策矩阵

| 维度 | D_JSON+SM | G(+检索层) | E_Full |
|------|-----------|-----------|--------|
| 绝对supported | **54** | 50 | 20 |
| supported rate | 56% | 52% | **100%** |
| Q23正确回答 | ✗ | ✓ | ✓ |
| 每题citations | 12.0 | 12.0 | 2.5 |
| 平均耗时 | 76s | 74s | **48s** |
| 答案覆盖面 | **广** | 广 | 窄 |
| 编造风险 | 中 | 中 | **极低** |

## 三个关键结论

### 1. E_Full (claim→evidence) 是质量最优但覆盖面最窄

- 100% supported rate，几乎零编造
- 但每题只有2-3个checks（vs旧方向12个），答案声明太少
- Q23是唯一正确回答"CAAC负责调查"的模式
- **适合：法律风险评估等要求零编造的场景**

### 2. D_JSON+SM (旧校验方向+JSON) 是覆盖面最优

- 54 supported最高，答案全面
- 但56% rate意味着近半声明无支撑
- Q19拒载从2/12→7/12（JSON模式直接贡献）
- **适合：需要全面回答的咨询场景**

### 3. 检索层(WRRF+AdaptiveK) 在JSON模式下无额外收益

- G_JSON_SM_Retrieval(50/96) vs D_JSON_SM(54/96)
- 检索层改进在旧校验方向下反而略降
- **建议：关闭WRRF和AdaptiveK，保持简单**

### 4. 阈值扫描：默认0.30无需调整

- threshold=0.20: 50/95 (52%) — 略优
- threshold=0.30: 50/96 (52%) — 默认值，几乎相同
- threshold=0.40: 30/86 (34%) — 过高，损失大
- LLM非确定性导致run间波动大于阈值差异
