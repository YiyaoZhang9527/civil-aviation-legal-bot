# 证据相关性门控实验

## 实验日期：2026-06-03

## 背景

消融v3 D配置（基线）的30题三层评测发现：
- 6题 faithful=0%，但没有警告用户
- Q8（结冰运行）0% faithful 但完全没有警告，看起来很专业
- Q13/Q20/Q21/Q23/Q24 引用了完全不相关的法规，用 `[合理推断]` 标记但力度不够

核心问题：检索到不相关证据时，LLM 用不相关法规拼凑出专业但不可靠的答案。

## 方案

在 SynthesisAgent 中加入证据相关性自评，让 LLM 在生成答案前评估证据是否真正回答了用户问题。

### v1: 评估+生成合并（单次调用）

在 synthesis prompt 中加 `relevance_assessment` 字段，LLM 先填评估再生成答案。

**结果（test30_20260603_064340）：**

| 忠实率 | 52.8% | 75.6% | +22.8pp |
| 幻觉率 | 2.8% | 0.0% | -2.8pp |
| F1 | 69.7% | 81.2% | +11.5pp |
| 拒答数 | 1 | 17 | +16 |

**问题：过度拒答。** 17/30 题被拒答，其中12题是错误拒答（Q7 85% faithful 也被拒了）。

- `none` 门槛提高到"完全不同法律领域"才触发
- `partial` 和 `sufficient` 都正常回答不加警告
- `none` 不再硬拒答，改为强警告

## v3: 先评估后生成（独立调用解耦）2026-06-03

### 实现
- config 加 `RELEVANCE_GATE_ENABLED = True`
- SynthesisAgent 加 `_assess_relevance()` 独立方法，轻量 prompt 只判断相关性
- `compose_answer()` 先调评估，`none` 时直接返回拒答模板，跳过生成
- synthesis prompt 完全恢复原样，零耦合

### 退步分析

7题退步中，实质退步的很少：
- Q2: 答案更诚实（承认证据不完整），评分降低但质量更好
- Q3: 答案合理（机长紧急权力+局方批准），评测器随机判0%
- Q12: 答案非常详细且正确引用了CCAR-135条款，33%评分偏低
- Q7/Q9: 仍在73-78%优秀范围，只是LLM生成随机性
- Q6: 本质持平(-5pp)
- Q29: 50%仍在可接受范围

### 结论

v3 是最终方案。Synthesis prompt 完全不变 → 不影响已有答案质量。独立评估调用 → 正确拦截 Q13/Q20 的不相关证据，且只有 1 个错误拒答。
