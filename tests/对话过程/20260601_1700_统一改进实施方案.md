```

  Step 1: 结构化输出改造            → llm.py + agents.py + types.py (~80行)

  Step 8: 阈值校准                  → config.py (~15行)
```

## Step 1: SynthesisAgent 结构化输出改造

**目标**: LLM输出JSON，每条claim带node_id引用。

**改动**:
- `llm.py` chat(): 加 `json_mode` 参数，传 `response_format={"type":"json_object"}`
- `agents.py` SynthesisAgent: 改prompt要求JSON输出，改用 `llm.json()` 调用
- `types.py` AnswerResult: 新增 `structured_claims` 字段

**输出Schema**:
```json
{
  "conclusion": "结论",
  "claims": [{"text": "声明", "node_ids": ["article:121.637"], "law_id": "CCAR-121"}],
  "legal_basis": "...", "conditions": "...", "risks": "..."
}
```

**验证**: 手动跑Q01/Q07/Q19，确认JSON可解析且claim有node_id

## Step 2: Set-Membership 确定性校验

**目标**: 纯代码校验，引用的node_id必须存在于检索结果集。~0ms延迟。

**改动**: agents.py `_verify_claim_node_ids()` (~15行)
**验证**: 构造伪造node_id，确认被过滤

## Step 3: Citation方向反转

**目标**: 从"每条evidence找claim"改为"每条claim验证是否被evidence支持"。

**改动**: citation.py `verify_with_cross_encoder()` 重写 (~55行)
- 批量构建 all_pairs = [(claim, ev.text)] 所有组合
- 一次 model.predict() 批量推理
- 每个claim取最高分evidence的分数判定status

**验证**: 对Q01运行，checks数量=claims数量（非evidence数量）

## Step 4: 答案修正/拒答

**目标**: 删除unsupported claims，全部删除则拒答。

**改动**: agents.py `_refine_answer()` (~25行)
**验证**: 构造全部irrelevant evidence，确认输出拒答模板

## Step 5: Adaptive-k 截断

**目标**: Level 1固定扩展30→15改为基于分数落差自适应。

**改动**: tree_retrieval.py 新增 `_adaptive_cutoff()` (~15行)，改2行截断逻辑
**验证**: Q06/Q18/Q22运行，确认噪声章节减少

## Step 6: WRRF + 法规名额分配

**目标**: 多query等权合并→WRRF加权。每法规最多3篇。

**改动**: agents.py `RetrievalAgent.retrieve()` 改为 `_wrrf_merge()` + `_cap_per_law()`
- 原始query w=1.0, 改写query衰减 0.7/0.5/0.3
- 每法规最多3篇，保底5篇总数

**验证**: Q14运行，确认CCAR-92证据不被空管规则稀释

## Step 7: 直通保底 + Reflexion修复

**目标**: 原始query top-4直通。Reflexion补搜不再清空law_hints。

**改动**: agents.py retrieve() (~15行) + _answer() 行940 `law_hints=[]` → `law_hints=plan.law_hints`
**验证**: Q18运行，确认补搜不引入空管规则

## Step 8: 阈值校准

**目标**: 遍历threshold 0.2-0.5，选最优值。

**改动**: config.py参数 + test_30questions.py扫描模式

## 风险

| 风险 | 缓解 |
|------|------|
| DeepSeek JSON mode不稳定 | config开关可回退自由文本 |
| Claim-NLI GPU显存 | batch上限60 pairs，超过分批 |
| Adaptive-k截断过激进 | min_k=5下限 |
| WRRF忽略改写query | 衰减有底限0.3 + 直通保底 |
| Reflexion保留law_hints过窄 | 补搜query含新法规关键词时追加 |
