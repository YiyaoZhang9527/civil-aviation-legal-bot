## 五、与之前分析的对比

### 之前报告（20260602_1800）的结论

> 根因：reranker 阈值降低引入噪声 + 多 query 合并抵消单 query 改进

### 本次更精确的结论

1. **reranker 阈值不是主因**。真正的主因是 Level 0 法规选择失败（34%）和通用条文挤占（24%）。reranker 只是在已经错误的候选集上做精排，无论阈值多少都救不回来。

2. **多 query 噪声确实是问题，但不是最大问题**。它贡献了 30% 的 unverifiable，与 Level 0 失败相当。

3. **之前的改进（法规多样性、扩展搜索）在单 query 测试中有效，但完整管线中没生效**——说明问题不在 Level 1/2，而在更上游：RewriteAgent 生成的 query 本身就把搜索方向带偏了。

## 六、修复优先级（修订）

### P0：Level 0 法规选择增强

当前 Level 0 只靠向量+BM25 对 130 部法规排序。小众法规（适航指令、无人机规则、禁带物品目录）语义距离远，排不进 top 30。

**方案选项**：
1. **法规名称关键词匹配**：query 中包含"无人机"→ 强制纳入 CCAR-92；"适航指令"→ 强制纳入适航指令规定
2. **扩大 Level 0 top_k**：30 → 50 或更高，确保小众法规不被遗漏
3. **CCAR 编号 hint 扩展**：当前只映射了 10 个 CCAR 编号，需要扩展到覆盖所有法规

### P0：通用条文降权

当前 Level 2 排序中，"目的和依据""适用范围""定义""运行合格审定"等通用条文总是排在前面，挤占了具体条款的名额。

**方案选项**：
1. **条文类型识别**：识别"目的""适用范围""定义""总则"类条文，在排序中降权
2. **按条文内容长度加权**：通用条文通常很短（几行），具体条款内容丰富，可用长度作为排序信号
3. **增加候选上限**：TREE_ARTICLE_CANDIDATES=30 → 50，让更多具体条款进入候选

### P1：RewriteAgent 质量控制

当前 RewriteAgent 会生成多个泛化 query（如"签派员"替代"签派员放行检查"），导致检索方向偏离。

**方案选项**：
1. **要求保留核心术语**：prompt 已改（20260602 改动），但效果待验证
2. **限制改写数量**：当前 MAX_SUBQUESTIONS=4，可以降到 2
3. **按法规分组合并**：不同 query 的结果按法规分组，避免跨法规噪声

### P2：禁带物品目录索引验证

Q21 拒答说明新增的禁带物品目录可能没有被索引重建覆盖。需要确认索引是否是最新的。
