# A2 关键词路由误杀 — 根因分析

**时间:** 2026-06-01
**现象:** 01:13 运行 30 题全部成功（evidence=12），06:04 运行 30 题全部失败（evidence=0）。配置完全相同。

## 根因

**A2 关键词路由的 Level 0 前置过滤把正确的法规踢掉了。**

01:13 运行用的是原始代码（无 A2 路由），向量模型自由召回，CCAR-121 正常命中。
06:04 运行加入了 A2 路由，`tree_retrieval.py` 的 `hint_law_ids` 在 Level 0 做前置过滤，
但 `_resolve_law_by_keywords()` 解析出的 hints 全部指向错误法规，导致 CCAR-121 被过滤。

## 具体案例

### Q01: "不在《运行规范》的机场是否可以去备降"
- 正确法规: `大型飞机公共航空运输承运人运行合格审定规则` (CCAR-121)
- A2 路由结果: `['外国公共航空运输承运人运行合格审定规则', '运输机场运行安全管理规定', '通用机场管理规定', '飞行模拟训练设备管理']`
- **CCAR-121 被完全排除**

### Q02: "飞机起飞前需要满足哪些燃油量要求"
- 正确法规: CCAR-121
- A2 路由结果: `['国内投资民航业...', '正常类飞机适航规定', '民用航空统计管理规定', '运输类飞机适航标准']`
- **CCAR-121 被完全排除**

### Q05: "飞行机组的值勤期限制是多少"
- 正确法规: CCAR-121
- A2 路由结果: `['一般运行和飞行规则']` (CCAR-91)
- **CCAR-121 被完全排除**

## 为什么路由错误

1. **`law_routes.json` 中 CCAR-121 的关键词缺少核心词**: 关键词是 "大型飞机公共航空运输"、"运行合格审定规则"、"承运人资质"、"航路批准"，但缺少 "备降"、"燃油"、"值勤期"、"签派"、"放行" 等实际用户会用的词。

2. **"备降" 在整个路由表中不存在**: 没有任何法规的路由规则包含 "备降" 这个词。

3. **通用词误匹配**: "运行规范" 匹配到了飞行模拟训练设备管理（因为它的关键词里也有"运行规范"）。"机场" 匹配到了运输机场运行安全管理规定。

4. **匹配阈值过于宽松**: `len(overlap) >= max(2, len(kw_tokens) // 2)` — 只需要 2 个 token 重叠就命中，导致大量误匹配。

5. **Level 0 前置过滤是致命的**: `tree_retrieval.py:203-207` 中，只要 `hint_law_ids` 非空且过滤后有结果，就直接替换 `law_set`。这意味着即使向量召回能找到正确法规，也被前置过滤干掉了。

## 关键代码路径

```
search_index_tree() (retrieval.py:357)
  → _resolve_law_by_keywords() (retrieval.py:269) — 关键词匹配，返回错误 hints
  → hint_law_ids 构造 (retrieval.py:486-494)
  → tree.search(hint_law_ids=...) (tree_retrieval.py:200)
  → Level 0 前置过滤 (tree_retrieval.py:203-207) — 踢掉正确法规
  → 后续章/条检索在错误法规内展开
  → Cross-Encoder 精排正确识别为不相关 → 全部过滤
  → evidence=0
```

## 修复方向

### 方案A（推荐）：前置过滤改为加权而非硬过滤

`tree_retrieval.py:203-207` 中，不要直接替换 `law_set`，而是对 hint 匹配的法规加分（boost），让向量召回结果仍然参与竞争。这样路由错误时不会被一刀切。

### 方案B：降低匹配阈值 + 增加 fallback

当前匹配阈值 `len(overlap) >= 2` 太宽松，导致误匹配。同时应在 `law_routes.json` 中补充 CCAR-121 的核心关键词（备降、燃油、值勤期、签派、放行等）。如果 hint 过滤后 law_set 为空或结果变少，应 fallback 到不过滤。

### 方案C（最快止血）：禁用 A2 前置过滤

临时在 `tree_retrieval.py` 中注释掉 Level 0 前置过滤逻辑，恢复纯向量召回行为，同时保留 hint_scores 对条级 RRF 的加权。
