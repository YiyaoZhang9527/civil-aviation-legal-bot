# 三题退步根因分析

**时间:** 2026-06-01
**退步题:** Q12 (12/12→7/12), Q14 (5/8→0/12), Q29 (11/12→2/12)

---

## 核心发现：树检索 Level 0→1/2 的「信号断裂」

树检索三级结构（法→章→条）存在一个结构性缺陷：**Level 1/2 完全不继承 Level 0 的法规排名信号**。

```
Level 0: 从129法中选出top 30，每法有RRF分数
         ↓
Level 1: 在全部30法内搜索章级，平等竞争，无视Level 0排名  ← 问题所在
         ↓
Level 2: 在top 15章内搜索条级，平等竞争，无视Level 0/1排名
         ↓
Cross-Encoder: 精排条级候选，无知于文章来自哪部法
```

这意味着：即使 Level 0 正确地把目标法规排在 #1，如果该法规的章级标题/摘要与 query 匹配度不如其他法规，Level 1 就会把其他法规的章节选出来，目标法规的条级文章被完全淘汰。

## 逐题根因

### Q12: CCAR-135 飞行员资质 (12/12 → 7/12)

**根因：CCAR-135 的 Dense 排名第 49 位，根本进不了 Level 0 top 30**

| 通道 | CCAR-135 排名 | 说明 |
|------|-------------|------|
| Dense | **第49位** (score=0.458) | TREE_TOP_LAWS=30，直接被排除 |
| BM25 | 第4位 (score=4.74) | 唯一救命通道 |
| RRF融合后 | 取决于BM25能否拉回来 | 信号微弱 |

**为什么 Dense 完全找不到 CCAR-135：**
- 法规标题："小型商业运输和空中游览运营人运行合格审定规则"
- Level 0 编码文本：标题 + 摘要 ≈ 120字
- **不含 "CCAR"、不含 "135"** → gte-large-zh 无法建立 "CCAR-135" 与该法规的语义关联
- query "CCAR-135部运营人的飞行员资质要求" 中的 "飞行员资质" 被泛化匹配到所有飞行员相关法规

**旧版 A2 如何修复这个问题：**
关键词路由 `{"keywords": ["135", "运行"], "hints": ["小型商业运输和空中游览运营人运行合格审定规则"]}` 直接命中，给 law_hints 注入正确法规名。

**当前版本为什么修不了：**
1. `_resolve_law_by_keywords()` 函数存在但**从未被调用**（死代码）
2. RewriteAgent (LLM) 可能生成也可能不生成正确的 law_hints
3. 即使生成 law_hints → hint_law_ids，在 Level 0 只是 +0.3 加分（boost），不改排名量级

### Q14: 无人机安全距离 (5/8 → 0/12)

**根因：Level 0 正确识别 CCAR-92，但 Level 1/2 被「安全距离」泛化语义劫持**

| 通道 | CCAR-92 排名 | 说明 |
|------|-------------|------|
| Dense | **#1** (score=0.577) | 完美 |
| BM25 | **#1** (score=8.37) | 完美 |
| Level 0 RRF | **#1** | 无可争议 |

Level 0 没有问题。问题出在 Level 1/2：

- query "无人机飞行的**安全距离**要求" 中 "安全距离" 是个跨法规通用概念
- Level 1 章级搜索时，「民用航空空中交通管理规则」的飞行间隔章节（专讲安全距离）得分可能高于 CCAR-92 的章节
- Level 2 条级搜索在错误法规的章节内展开
- Cross-Encoder 精排进一步确认了错误方向（空管规则文章关于"安全距离"的语义匹配度高）

**对比基线证据：**
- 基线：全部来自 CCAR-92（92.631 视距内运行, 92.617 设备要求, 92.625 飞行区域限制）→ 5/8 supported
- 当前：混入空中交通管理规则 + 机场运行安全管理规定 → 0/12 supported

**为什么旧版不退步：**
A2 关键词路由 `{"keywords": ["无人机"], "hints": ["民用无人驾驶航空器运行安全管理规则"]}` 会强制 hint_law_ids 包含 CCAR-92，Level 0 给 CCAR-92 额外 +0.3 boost。但更重要的是旧版的 Level 0 后置过滤会踢除非匹配法规——这在当前版本被移除了。

### Q29: 航空器所有权 (11/12 → 2/12)

**根因：Level 0 正确，但 Level 1 完全被民用航空法接管**

**Level 0 结果（完全正确）：**
```
#1 权利登记条例 score=0.125000  ← 目标法规，远超第二名
#2 航空器适航指令 score=0.088346
...
#17 民用航空法 score=0.047619  ← 排名很低
```

**Level 1 结果（完全错误）：**
```
#1 民用航空法 score=0.121324  ← 从Level 0的#17跳到Level 1的#1！
#2 民用航空法 score=0.121324
...
权利登记条例：完全不在top 15中！
```

**为什么权利登记条例在 Level 1 消失：**
- 权利登记条例实施办法是一个小法规（可能是扁平结构，章级节点少或无）
- Level 1 章级搜索时，民用航空法的 "民用航空器" 相关章节与 query "民用航空器所有权" 有更高的 Dense/BM25 分数
- 权利登记条例的章节标题可能是 "申请"、"审查"、"发证" 等程序性标题，与 "所有权取得转让" 的语义匹配度低

**旧版如何修：**
A2 硬路由会把 hint_law_ids 设为仅包含权利登记条例，Level 0 后置过滤只保留该法规，民用航空法直接被踢除。

## 共同模式

| 维度 | Q12 | Q14 | Q29 |
|------|-----|-----|-----|
| Level 0 是否正确 | ❌ (CCAR-135排49) | ✅ (CCAR-92排#1) | ✅ (权利登记排#1) |
| Level 1/2 是否正确 | ❌ | ❌ | ❌ |
| A2 路由能否修复 | ✅ | ✅ | ✅ |
| 退步根因 | Dense无法匹配CCAR编号 | 安全距离泛化劫持 | Level 0→1信号断裂 |

**三道题的共同根因：废弃 A2 关键词路由后，没有替代机制在 Level 1/2 传递法规级偏好信号。**

## 代码层面的具体问题

### 问题 1: law_id 不在 Level 0 编码文本中
```python
# tree_retrieval.py _node_text()
def _node_text(node, parent_title="", grandparent_title=""):
    parts = [grandparent_title, parent_title, node.title, node.summary or ""]
    # ← 缺少 law_id（如 CCAR-135、CCAR-92）
```

CCAR-135 的标题是 "小型商业运输和空中游览运营人运行合格审定规则"，Level 0 编码文本不含 "CCAR" 或 "135"。

### 问题 2: Level 1/2 无条件继承 Level 0 排名
```python
# tree_retrieval.py search() Level 1
ch_law_mask = np.isin(self._chapter_to_law, np.where(law_mask)[0])
# ← law_mask 是二值的（在/不在 top 30），丢失了 Level 0 的排名分数
ch_vec = self._filtered_vector_search(query, self._chapter_emb, ch_law_mask, top_chapters)
ch_bm25 = self._filtered_bm25_search(query, self._chapter_bm25, ch_law_mask, top_chapters)
chapter_set = self._rrf_union(ch_vec, ch_bm25, top_chapters)
# ← _rrf_union 只融合 Dense+BM25，Level 0 分数完全丢失
```

### 问题 3: _resolve_law_by_keywords 是死代码
```python
# retrieval.py 定义了函数但从未调用
def _resolve_law_by_keywords(query, law_hints): ...  # 269-314行
# search_index_tree() 和 _tree_search() 中均无调用
```

## 修复方案（按优先级）

### 修复 1: law_id 加入 Level 0 编码文本（解决 Q12）
```python
# tree_retrieval.py _node_text()
def _node_text(node, parent_title="", grandparent_title="", law_id=""):
    parts = [law_id, grandparent_title, parent_title, node.title, node.summary or ""]
    return " ".join(p for p in parts if p)
```

这样 "CCAR-135" 会出现在 Level 0 文本中，Dense 和 BM25 都能匹配。

### 修复 2: Level 0 分数传播到 Level 1/2（解决 Q14、Q29）
在 Level 1 章级搜索后，根据章节所属法规的 Level 0 排名，给章节分数加一个法规先验权重。

### 修复 3: 恢复简化版关键词路由（兜底）
不是恢复旧的 A2 硬路由（已知误杀），而是一个轻量的 "CCAR 编号识别"：
- 正则从 query 中提取 CCAR-\d+ 编号
- 直接映射到对应法规（确定性，零 LLM）
- 作为 Level 0 的额外 boost 信号

这三个修复互相独立，可以逐步实施。
